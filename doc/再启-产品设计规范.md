## 一、品牌与定位

### 产品名称
**再启** — AI 再婚决策引擎

### 产品定位
再启 — 你的 AI 再婚决策顾问

### 核心理念
- 这不是交友平台，是**决策引擎**
- 交友是数据输入，决策是核心输出
- AI 不是工具，是**全程的理性大脑**

### Slogan
- 再启 — 让再婚少走弯路
- 再启 — 你的 AI 再婚决策顾问
- 第二次，更懂选择

## 四、S1：基础用户系统

### 4.1 注册与登录

| 终端 | 登录方式 |
|------|---------|
| 微信小程序 | 微信授权手机号免密登录（主要方式） |
| H5 网页版 | 手机号 + 短信验证码 |
| Web 管理后台 | 手机号 + 密码（仅管理员） |

**注册流程：**
1. 用户进入小程序 → 微信授权获取手机号
2. 首次进入 → 引导填写基础资料（昵称、性别、出生年份、所在城市）
3. 设置登录密码（用于后续 Web 端管理后台登录）
4. 自动创建用户 → 进入基础会员状态

**认证机制：**
- JWT token：HMAC-SHA256，存储于小程序 storage
- Redis 记录 refresh_token，实现无感续期
- 敏感操作（充值）要求重新验证身份

### 4.2 个人资料

**基础信息（所有人可见）：**
- 头像（最多 3 张，至少 1 张）
- 生活照（最多 6 张）
- 昵称、性别、出生年份、所在城市
- 个人简介
- 婚姻状况（离异/丧偶/未婚）、有无子女、子女抚养情况
- 职业、学历、年收入范围

**5 维匹配数据（填写但仅用于算法，推荐结果可见）：**
| 维度 | 说明 |
|------|------|
| 价值观 | 信仰/人生观重要度 |
| 生活方式 | 作息习惯、兴趣爱好、社交倾向 |
| 经济观念 | 消费观、储蓄观、房产情况 |
| 家庭观念 | 再婚后的家庭分工、与对方子女相处态度 |
| 感性态度 | 情感表达方式、对亲密关系的期待 |

### 4.3 认证体系

| 认证类型 | 触发条件 | 前端展示 |
|---------|---------|---------|
| 手机号认证 | 注册即完成 | 显示「已认证手机」 |
| 身份认证 | 首次充值时（微信支付实名校验） | 显示「已实名认证」 |
| 真人认证 | 可选——上传手持身份证，后台人工审核 | 显示「真人认证」 |

### 4.4 隐私设置

- **在线状态开关**：用户可设置是否显示在线状态
- 关闭后：自己的在线状态对其他用户不可见
- 关闭后：自己也看不到其他人的在线状态
- 默认：开启
- 不影响聊天消息收发，仅影响在线标识展示

### 4.5 资料审核机制

- 头像/照片：后台人工或自动审核（敏感内容检测）
- 身份认证：微信支付实名信息一致性校验
- 敏感字段修改（婚姻状况、子女等）：标记「待审核」，后台可见

## 六、S3：推荐与匹配

### 6.1 5 维匹配算法

**匹配分 = 5 个维度加权平均（每维 20 分，总分 100 分）**

每个维度通过选择题收集用户数据（选项对应分值）。匹配时计算两人在该维度的差距绝对值 → 差距越小得分越高。默认各维度权重 20%。

**示例（生活方式维度）：**
```
问题："你平时的作息习惯是？"
  A) 早睡早起（5分）  B) 规律作息（4分）
  C) 偶尔熬夜（2分）  D) 夜猫子（0分）

用户甲选 A(5分)，用户乙选 C(2分)
→ 差异 = |5-2| = 3 → 得分 = 20 - 3 = 17分
```

### 6.2 混合推荐算法

**最终推荐分 = 5 维基础匹配分 × 60% + 行为信号分 × 40%**

**行为信号分构成：**

| 信号 | 分值范围 | 说明 |
|------|---------|------|
| 资料完整度 | 0~10 分 | 填写信息越完整，推荐越靠前 |
| 活跃度 | 0~15 分 | 近 7 天登录频率（每日递减权重） |
| 照片质量/张数 | 0~5 分 | 鼓励上传真实照片 |
| 互动热度 | 0~20 分 | 被查看资料频率、被点赞数、动态互动量 |
| 响应率 | 0~10 分 | 收到聊天请求后回复比例和速度 |
| 身份认证 | +10 分 | 已实名认证额外加分 |
| 负向信号 | -10~-50 分 | 被举报次数、频繁修改资料、异常登录 |

**推荐策略：**
- **冷启动（新用户前 7 天）**：5 维匹配分 + 资料完整度 + 认证状态优先，推荐活跃认证齐全的用户
- **成熟期**：行为信号权重逐步提升，根据用户查看行为倾向做倾斜
- **互动加权**：互相点赞 → 匹配分额外 +5 分
- **拒绝反馈**：点击「不感兴趣」→ 30 天内不再推荐同类偏差过大的用户

**推荐池刷新：**
- 基础会员：每日早 8 点刷新 N 条推荐，当日用完即止
- 高级会员：实时刷新，每次去重已查看的

**排除逻辑：**
- 不推荐已屏蔽/举报过的用户
- 不推荐同性别（除非用户设置开启）
- 不推荐已聊天并标记「不合适」的

## 七、S4：社交系统

### 7.1 聊天系统

- 协议：WebSocket + JWT 认证（query param 携带 token），Spring WebSocket 3.2.x
- 仅高级会员可用
- 一对一文字消息（支持文本、图片）
- 聊天界面顶部提示：「系统内聊天便于AI决策报告分析，建议将交流保留在此」
- 基础会员点击聊天 → 引导充值页面
- 已读/未读标识
- 消息撤回（2分钟内）
- 屏蔽/举报对方

### 7.2 动态系统

- 仅高级会员可发布动态；所有认证用户可查看
- 文本（最多 500 字）+ 图片（最多 9 张）
- 可见范围：全部会员 / 仅高级会员
- 点赞：显示点赞数
- 评论：文本评论，按时间正序排列
- 作者可删除自己的评论
- 后台可删除违规动态及评论

## 八、S5：AI 决策引擎

### 8.1 完整决策报告（高级会员）

**触发条件：** 双方互发 ≥ 10 条消息后，可点击「生成决策报告」
**免费额度：** 每月 5 次，超出 ¥9.9/次，月初重置

**报告内容：**

① **5 维匹配分析概览**
```
价值观     ████████░░ 82分  高度匹配
生活方式   ██████░░░░ 65分  存在差异
经济观念   ███████░░░ 73分  良好
家庭观念   █████████░ 91分  非常匹配
感性态度   ████████░░ 80分  匹配
──────────────────
总分：     ███████░░░ 78分
```

② **风险提示**
- 用户自行填写的风险字段（婚姻状况、子女抚养权等）
- 系统自动标记的行为风险（资料频繁修改、异常登录等）
- 后台管理员标注的风险

③ **AI 深度分析（基于聊天对话）**
- 分析双方聊天记录，提炼沟通模式、情绪、价值观流露
- 给出 AI 视角的个性分析

④ **关系匹配判断**
- 综合评估是否适合走向婚姻
- 分档：非常合适 / 有一定潜力 / 需要更多了解 / 建议谨慎

⑤ **行动计划清单**
- 针对分析结果给出可操作建议
- 示例：「你们的消费观念差距较大，建议进一步沟通」

### 8.2 单次 AI 深度分析（¥9.9，基础会员可用）

简化版报告，仅包含：
- 5 维匹配分析概览
- 风险提示
- **不包含**聊天分析和决策判断

### 8.3 再婚 Readiness 评估

独立入口，所有人可用：
- 问卷 10-15 题（上一段婚姻反思、再婚期待与担忧、子女接受度、经济准备）
- AI 生成「再婚 readiness 评估报告」
- 可作为引流手段

### 8.4 对话辅助（高级会员）

- 消息建议：AI 推荐措辞（敏感话题时）
- 话题引导：根据资料和聊天历史推荐话题
- 风险提示：聊天中出现需警惕的信号时 AI 实时提示
- 均为可选功能，用户主动点击使用，不自动介入

### 8.5 AI 服务架构

独立部署的服务，通过 WebClient 调用 DeepSeek API。与主服务解耦，AI 计算不影响主流程。
